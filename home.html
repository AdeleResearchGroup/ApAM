APAM is an extensible framework for designing, developing and executing dynamic and resilient service-based applications.


Built on top of OSGi, APAM simplifies the design, development and management of applications: dynamism, composition, resilience are automatically managed.



Design

APAM components are designed following the APAM component model, which defines component types (specification, implementation and instance) at different abstraction levels (conception, development, execution).

Development

APAM components are developed as POJOs (Plain Old Java Objects), which implement only the component business logic. Various aspects, like dynamism, resilience or distribution, are automatically managed by the APAM runtime.

Composition

APAM allows defining composite components. Components can be then herarchically structured like black-boxes, white-boxes or gray-boxes. Composites allow defining and managing applications in a finer way.



Execution

The APAM runtime supports the concurrent execution of multiple applications. It provides dynamic discovering, injection, deployment of required services and devices, managing then the life cycle of components and applications.

Dynamism

APAM provides the dynamic aspect to components and applications: applications are built dynamically via the on-demand resolution of service dependencies.

Resilience

APAM provides the resilience aspect to applications: applications adapt dynamically to the evolution of their execution context, and continue working.



Extensibility

APAM provides an extension mechanism via managers. A manager define and manage specific aspects of components and applications. Currently, APAM provides dependency and property managers.

Distribution

Through its extension mechanism, APAM provides a distribution manager that controls the discovery and communication of remote APAM machines and services.
